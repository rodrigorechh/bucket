AMAZON S3: Simple Storage Service 

    -Criado para armazenar arquivos
    -O espaço de armazenamento é ilimitado, mas o limite máximo por arquivo é 5 tb
    -Geralmente você paga por GB de uso, mas depende do seu plano. Também vai depender da zona e quão rápido você quer acessar o arquivo
    
    -A lógica do serviço é a seguinte:
        -Você cria um bucket e dentro do bucket vc armazena seus arquivos
        -O nome da bucket tem que ser exclusivo.
            -Tem a mesma regra de criar um nome pra DNS, não pode usar espaço ou caracteres especiais nem mais de 63 caracteres.
                -Pq internamente cria um subdomínio
            -O nome gera um registro no DNS. Então vc vai ganhar a sua URL
                -estrutura: https://s3-us-west-2.amazonaws.com/nome-bucket/nome-folder
                                    us-west-2: nome da região, nome-bucket e nome-folder

    -Disponibilidade(Avaliability):99.9%
    -Durabilidade/Integridade(Durability):99.99999999999%

    -Versionamento: Quando os arquivos são alterados, eles podem ser versionados
        -Vc habilita o versionamento abrindo a bucket, na aba properties ativar a opção Versioning
        -Qnd essa opção é marcada, na aba overview do bucekt aparece a opção de mostrar o versionamento, que mostra as versões anteriores do arquivo
        -Ao ativar o versionamento não é possível desmarcar a opção, apenas marcar a opção de suspender, aí vai parar ed versionar nas futuras alterações. 
        -url passa a ter a versão caso seja ativada, assim que a aws diferencia uma da outra.
            -ex: https://s3-us-west-2.amazonaws.com/nome-bucket/nome-folder?versionId=12333
            -Inclusive, se ele for public e versionado, vai continuar sendo public
        -Caso o arquivo seja deletado ele continua na aba "show". Vc precisa deletar o histórico de deleta da aba "show" para o arquivo ser restaurado
            -Isso só ocorre caso vc delete o arquivo, se vc deletar uma versão antiga versionada, vai deletar de verdade.

    Componentes:
        -Object base: 
            -Quando upload é feito, a aws primeiro lê o arquivo, depois escreve.
            -Gera uma key com o nome do arquivo
            -value é os dados do arquivo
            -version é a versão
            -metadata: armazena de dados como tags, locais
            -ACL: define se o arquivo é publico ou privado

    Tier(classe): Ao publicar um arquivo é necessário escolher o tier dele.
	S3 Standart: Armazenamento com alta performance, para dados acessados com frequência
	S3 Intelligent-Tiering: promove a realocação automática de arquivos entre diferentes categorias de armazenamento. Para isso, ele usa dados de acesso monitorados pela própria plataforma, garantindo a guarda de arquivos pelo menor preço possível.
		obs: Existe uma taxa de monitoramento a cada mil arquívos. Caso tenha muitos arquivos pequenos pode acabar nao valendo a pena pois a 	economia por aquivo poderá ser menor que a taxa de monitoramento cobrada. Já se o tamanho
		dos arquivos forem maíores a economia pode ser maíor e a taxa de monitoramento vai ser a mesma.
	S3 Standart IA(Infrequent Access): Vc paga menos pelo armazenamento, mas vc paga toda vez que acessa
	S3 One Zone IA(Infrequent Access): Paga menos que no passo anterior, é replicado em 1 availability zone
		-os dados armazenados nessa categoria de armazenamento serão perdidos em caso de destruição da zona de disponibilidade.
	S3 clacier Instant Retrievals Paga muito pouco pelo armazenanento e bastante pelo acesso.
		-Recomendado para arquivos que serão acessados poucas vezes (uma vez a cada 3 meses) mas quando serão acessados precisam de acesso inediato.
	S3 Glacier Flexíble Retrieval e Glacier Deep Archives O gb de armazenamento são os mais baratos, mas demoram horas pra serem acessados.
    https://aws.amazon.com/pt/s3/storage-classes/

    Tipos de pagamento: vc pode configurar por aruivo salvo na bucket
        -Storage: pagamento por giga armazenado
        -Request: pagamento por acesso(não existe apenas no tier standart)
        -Storage Managment Price: pagamento por armazenamento de metadados
        -DTP: Preço de transferencia de uma região para outra caso necessário. Se subir uma instancia no EC2 e ela tiver em uma região, porém salvar no bucket que estiver em outra região, a taxa DTP é cobrada.
        -Transfer Aceleration(CDN/Cloudfront): É quando o arquivo do bucket é replicado em diversas Availability Zones
